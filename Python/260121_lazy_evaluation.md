# Lazy Evaluation
오늘은 게으른 연산(Lazy Evaluation)에 대해 배운 내용을 정리해본다.
## 게으른 연산이란?
게으른 연산(Lazy Evaluation)은 필요한 시점까지 계산을 미루는 프로그래밍 기법이다. 즉, 값이 실제로 필요할 때까지 계산을 수행하지 않는다는 의미이다. 이는 메모리 사용을 최적화하고, 불필요한 계산을 피함으로써 성능을 향상시킬 수 있다.

map()함수를 사용하다가 보면, 이 함수가 즉시 결과를 반환하지 않고, 대신에 '맵 객체'를 반환하는 것을 알 수 있다. 이 맵 객체는 실제로 값을 계산하지 않고, 필요할 때마다 값을 생성하는 역할을 한다.

```python
# 예시: map() 함수와 게으른 연산
def square(x):
    print(f"Calculating square of {x}")
    return x * x
numbers = [1, 2, 3, 4, 5]
squared_numbers = map(square, numbers)  # 즉시 계산되지 않음
print("Map object created.")
print(squared_numbers)  # <map object at ...>
# 실제로 값을 사용하려면 반복문이나 list()로 변환해야 함
for num in squared_numbers:
    print(num)  # 이 시점에서 실제로 계산이 수행됨
```

## 장점
1. **메모리 효율성:** 모든 값을 한 번에 계산하지 않고, 필요한 값만 계산하므로 메모리 사용량이 줄어든다.
2. **성능 향상:** 불필요한 계산을 피할 수 있어, 성능이 향상될 수 있다.
3. **무한 시퀀스 처리:** 무한한 데이터 스트림을 다룰 때 유용하다. 필요한 만큼만 값을 생성할 수 있다.

## 단점
1. **디버깅 어려움:** 계산이 지연되기 때문에, 디버깅이 어려울 수 있다.
2. **예측 불가능한 성능:** 계산이 지연되면서, 성능이 예측하기 어려워질 수 있다.

## 결론
게으른 연산은 메모리 효율성과 성능 향상에 큰 도움이 될 수 있는 유용한 기법이다. 그러나 디버깅의 어려움과 예측 불가능한 성능 문제를 고려해야 한다. 파이썬에서 제공하는 다양한 게으른 연산 도구들을 활용하여 효율적인 코드를 작성할 수 있다.
