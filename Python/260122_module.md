# Module
오늘은 모듈에 대해 수업을 진행했다. 이에 따라 모듈에 대해 정리해본다.
## 모듈
모듈은 관련된 함수, 클래스, 변수들을 하나의 파일(.py)로 묶은 것이다. 모듈을 사용하면 코드의 재사용성과 유지보수성이 향상된다.

한줄 정리 : 한 파이로 묶인 변수와 함수의 모음 
### 모듈 활용
### 1. Import 문 활용
모듈을 사용하려면 `import` 문을 사용하여 모듈을 불러와야 한다.
```python
import 모듈이름
```
**Caution**

같은 이름의 함수가 여러 모듈에 있을 경우, 모듈 이름을 명시하지 않으면 충돌이 발생할 수 있다.
```python
import math
print(math.sqrt(16))  # 4.0
```
단, 명시적이지만 코드가 길어질 수 있다는 단점이 있다.

### 2. from ... import 문 활용
특정 모듈에서 필요한 함수나 클래스만 선택적으로 불러올 수 있다.
```python
from 모듈이름 import 함수이름, 클래스이름
from math import sqrt
```
**Caution**

이 방법은 코드가 간결해지지만, 정의된 모듈의 위치를 알기 어려워 명시성이 떨어진다.

사용자가 선언한 변수 또는 함수와 겹치게 돼 모듈에서 정의한 동작이 이루어지지 않을 수 있다.
```python
from math import sqrt
print(sqrt(16))  # 4.0
def sqrt(x):
    return "This is a custom sqrt function."
print(sqrt(16))  # This is a custom sqrt function.
```

**Caution2**
모든 요소를 한번에 불러오는 `*` 사용은 권장되지 않는다. 이는 코드의 명시성을 떨어뜨리고, 디버깅을 어렵게 만들 수 있다.
```python
from math import *
```
### 3. as 키워드 활용
다수의 모듈을 사용할 때, 이름 충돌을 피하기 위해 `as` 키워드를 사용하여 모듈이나 함수에 별칭을 지정할 수 있다.
```python
import 모듈이름 as 별칭
import numpy as np
```

## 파이썬 라이브러리
### 패키지
연관된 모듈들을 디렉토리 구조로 묶은 것이다. 패키지는 모듈의 조직화를 돕고, 네임스페이스를 제공한다.
```plaintext
my_package/
    module1.py
    module2.py
```

### 표준 라이브러리
파이썬에 기본적으로 포함된 다양한 모듈과 패키지들의 집합이다. 
```python
import os  # 운영체제 관련 기능
import sys  # 파이썬 인터프리터 관련 기능
import math  # 수학 관련 기능
```
별도 설치 없이 바로 사용할 수 있다는 장점이 있다.

### 외부 패키지(Third-Party Packages)
필요한 기능을 사용하기 위해 직접 설치해서 쓰는 패키지
- 전세계 개발자들이 만든 다양한 기능의 패키지들이 존재
- `pip` 명령어를 사용하여 설치
```bash
pip install 패키지이름
pip install SomePackage         # 최신 버전 설치
pip install SomePackage==1.0.0  # 특정 버전 설치
pip install SomePackage>=1.0.0  # 최소 버전 설치
```
다양한 패키지 버전이 존재하고, 개발시 호환성 문제를 일으킬 수 있으므로 주의가 필요하다.
#### Tip. requirements 파일
프로젝트에서 사용하는 외부 패키지와 그 버전을 명시한 파일. `pip`를 사용하여 한 번에 필요한 패키지를 설치할 수 있다. 협업 시 개발 환경을 통일하는 데 유용하다.
```plaintext
# requirements.txt

SomePackage==1.0.0
AnotherPackage>=2.0.0
```
```bash
pip install -r requirements.txt
```
