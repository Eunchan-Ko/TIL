## 📋 학습 원칙 (Study Rules)
- **1 Day, 1 Problem:** 매일 최소 1개 이상의 알고리즘 문제를 풀고 기록합니다.
- **Deep Dive:** 단순히 코드를 짜는 것에 그치지 않고, 왜 이렇게 풀었는지 사고 과정을 기록합니다.

## ✍️ 문서 생성 규칙 (Formatting)
모든 문서는 일관성을 위해 아래 규칙을 따릅니다.

### 1. 파일명 규칙
- **형식:** `yymmdd_title.md` (또는 `.py`)
- **예시:** `260120_binary_search.md`, `260120_boj_10825.py`

### 2. 태그(Tag) 활용
문서 상단에 아래와 같이 태그를 삽입하여 관리합니다.
```markdown
---
date: 2026-01-20
tags: [Python, Algorithm, BFS]

---
```
